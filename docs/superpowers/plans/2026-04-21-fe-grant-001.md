# FE-GRANT-001 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the minimum frontend slice for `GRANT`: public grant list and detail pages, an applicant grant apply page wired to the real backend APIs, and explicit prerequisite UX that makes the linked conference requirement visible instead of burying it in generic errors.

**Architecture:** Build a dedicated `frontend/src/features/grant` module parallel to `conference`, not an extension of `frontend/src/features/conference`. Reuse only shared patterns: API wrapper shape, transport/domain mappers, provider split (`fake` for tests, `http` for runtime), and existing page/layout CSS conventions. Add three new page routes (`/grants`, `/grants/:slug`, `/grants/:slug/apply`) and a grant-specific apply form that clearly distinguishes grant applications from conference applications while still fitting the current visual system.

**Tech Stack:** React, React Router, TypeScript, Vite, Vitest, Testing Library, Axios

---

## File Structure

- `frontend/src/api/grant.ts`
  New HTTP request helpers for grant list/detail/form/draft plus shared applicant draft and submit calls.
- `frontend/src/features/grant/types.ts`
  Domain types for grant list/detail/application/form plus the provider contract.
- `frontend/src/features/grant/grantMappers.ts`
  Transport-to-domain and domain-to-transport mappers for the new grant contract.
- `frontend/src/features/grant/grantProvider.ts`
  Runtime selection between fake and HTTP provider, mirroring the existing conference pattern.
- `frontend/src/features/grant/httpGrantProvider.ts`
  Real-API implementation wired to `BE-GRANT-001`.
- `frontend/src/features/grant/httpGrantProvider.test.ts`
  Unit tests for transport mapping and 404 / 409 / 422 error handling.
- `frontend/src/features/grant/fakeGrantProvider.ts`
  Local fake state used only by tests so page tests stay deterministic.
- `frontend/src/features/grant/GrantListCard.tsx`
  Card component for `/grants`.
- `frontend/src/features/grant/GrantApplyForm.tsx`
  Grant-specific draft/edit/submit form.
- `frontend/src/pages/Grants.tsx`
  Public grant list route.
- `frontend/src/pages/GrantDetail.tsx`
  Public grant detail route with explicit conference prerequisite messaging and apply CTA.
- `frontend/src/pages/GrantApply.tsx`
  Applicant grant draft/edit/submit page.
- `frontend/src/pages/GrantApply.test.tsx`
  Page-level tests for unauthenticated state, prerequisite messaging, draft creation, draft hydrate, and submit.
- `frontend/src/pages/Conference.css`
  Reuse as the base layout stylesheet; only extend if grant-specific states need a few additional selectors.

## Scope Guard

- Do not change `frontend/src/features/conference` into a mixed conference/grant module.
- Do not add applicant dashboard or result pages.
- Do not add post-visit report UI.
- Do not redesign global navigation.
- Do not “fix while here” unrelated conference page behavior.
